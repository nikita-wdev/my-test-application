Инструкция по поднятию фронтенда и бэкэнда лежит внутри каждого проекта в readme.md

По бэкэнду:
 - сделал обычный express app с простенькой структурой
 - с роутингом решил не заморачиватся под два API метода, но можно было сделать динамический CRUD routing.
 - в коде оставил комментарии TODO где можно было бы сделать более кошерно
 - Добавил пакет husky, для примера работы с git, чтобы можно было вешать хуки на commit/push, к примеру npm run lint на commit и запуск тестов на push.

 По Фронтеду:
  - Структура с нуля под reactjs app никогда не делал, по этому сделал простенький app на две страницы
  - Таблицу сделал без использования сторонних пакетов, но можно было бы заюзать datatable, react-grid и т.д которые легко настраивались бы конфигам,
    и имели встроенные функции сортировки/фильтрации/пагинации/поиска и т.д
  - orderBy - сделал по одной из колонок, не вижу смысла тут делать мульти ордер бай
  - Для пагинации использовал готовый пакет заточеный под react-app, т.к не вижу смысла изобретать велосипед.
  - При работе с таблицей добавляю query параметры, чтобы можно было перейти по ссылке с фильтрами/страницами/сортировками
  - Не использовал linter для react-app.
  - В таблице колонку location не фильтровал по уникальному значению.
  - в App добавил nav и footer, но не выносил их в лаяут
